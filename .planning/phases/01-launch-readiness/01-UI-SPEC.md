| Token | Value | Usage |

| sm | 8px | Tight element spacing, input padding |
| md | 16px | Default element gap, form fields |

Exceptions:
- Touch targets on URL input submit button: minimum 44px height
- Stat card numbers: allow 40px line-height for display numerals (not a spacing token, a type token)

## Typography

| Role | Size | Weight | Line Height | Font | Tailwind Class |
|------|------|--------|-------------|------|----------------|
| Display | 48–60px (text-5xl/6xl) | 700 (bold) | 1.1 | Syne | `font-display text-5xl font-bold leading-none tracking-tight` |
| Heading | 28–36px (text-3xl/4xl) | 700 (bold) | 1.2 | Syne | `font-display text-3xl font-bold leading-tight` |
| Body | 16–18px (text-base/lg) | 400 (regular) | 1.6 | DM Sans | `font-body text-lg font-normal leading-relaxed` |
| Label / Caption | 14px (text-sm) | 400 (regular) | 1.4 | DM Sans | `font-body text-sm font-normal` |

**Font weights: 2 only — 400 (regular) and 700 (bold).** Display and Heading are both bold; visual contrast between them comes from size difference (48–60px vs 28–36px), not weight. Label/Caption uses 400 at 14px — the smaller size differentiates it from body without a separate weight.

Notes:
- Stat numbers (sektion 3: "64%", "1 av 10", "0 kr"): text-5xl, font-bold, accent color, Syne
- Code examples in checklist: `font-mono text-sm` — system monospace stack (JetBrains Mono not required)
- Mobile: Display scales to text-4xl on small screens (`text-4xl md:text-5xl lg:text-6xl`)
- Body text max-width: prose `max-w-2xl` — matches landing-page-proposal.md's "max-w-4xl for textsektioner"

**Source:** landing-page-proposal.md §Typografi, RESEARCH.md §Pattern 6 (Syne + DM Sans). Weights reduced from 4 to 2 per checker fix (BLOCK Dimension 4).

## Interaction Contract

### URL Input (3 instances)

| Instance | Location | Behavior |
|----------|----------|---------|
| Primary | HeroSection | Submits directly — calls `analyze(url, city)` in ToolSection via scroll + anchor. Use `<a href="#analysera">` CTA that scrolls down, then user re-enters in ToolSection. OR: Hero has its own input that scrolls to `#analysera` and pre-fills it. |
| Secondary | ChecklistSection | Soft CTA button: "Analysera din sida" → `href="#analysera"` scroll. No live input here. |
| Primary (canonical) | ToolSection `id="analysera"` | The only live scan trigger. Connected to `analyze()` from `useAnalysis`. |

**Decision (from RESEARCH.md):** Use anchor `#analysera` for Hero and Checklist, not duplicate live inputs. This avoids race conditions and stale state. Hero CTA = scroll button. Checklist CTA = scroll button. ToolSection = live input.

### Scroll-reveal
- All sections below the fold use the `R` wrapper with staggered `delay` props (0, 1, 2...)
- HeroSection does NOT use `R` — it renders at full opacity immediately
- Threshold: 0.01 (fires very early), rootMargin: '20px 0px 0px 0px'

### Animated stat counters (ProblemSection)
- AnimNum fires once per number when scrolled into view (threshold 0.5)
- Duration: 1200ms, ease-out cubic
- Values: 64 (suffix "%"), then text "1 av 10", then 0 (suffix " kr")
- "1 av 10" does not animate (non-numeric) — render static

### ExplainerSection animation
- On scroll into view (threshold 0.2): setTimeout 600ms → `setShowAI(true)`
- Left (Google panel): fades to `opacity-30 blur-sm`
- Right (AI panel): slides in from `translate-x-4 opacity-0` to `translate-x-0 opacity-100`
- Transition duration: 700ms

### PremiumCTA states
| State | Display |
|-------|---------|
| Hidden | `state !== 'done'` or `enhancedReport === null` |
| Visible — idle | Email input + submit button |
| Submitted | "Vi hör av oss!" confirmation message. No spinner needed. |

### Scan flow states (ToolSection)
| State | Display |
|-------|---------|
| `idle` | UrlInput only |
| `scanning` | Progress component (existing) |
| `done` | EnhancedReport + PremiumCTA below |
| `error` | Error message (see Copywriting §Error) |

## Animation Contract

| Effect | Trigger | Duration | Easing | Applies to |
|--------|---------|----------|--------|-----------|
| Scroll reveal (`.rv → .rv.vis`) | IntersectionObserver (threshold 0.01) | 650ms | `cubic-bezier(.16,1,.3,1)` (spring) | All sections below fold, individual cards |
| Stagger delay | Per child `delay` prop | 0.1s × n | same | Checklist items, stat cards, mistake cards |
| Animated counters | IntersectionObserver (threshold 0.5) | 1200ms | `1 - (1-p)^3` ease-out | Stat numbers in ProblemSection |
| AI panel reveal | IntersectionObserver + 600ms delay | 700ms | `ease-in-out` | ExplainerSection right panel |
| Google panel fade | Same trigger as above | 700ms | `ease-in-out` | ExplainerSection left panel |

No CSS keyframe animations, no third-party animation libraries. All via IntersectionObserver + Tailwind transition utilities + CSS custom properties.
